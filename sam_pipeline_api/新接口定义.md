## 说明

- 请求里的 `camera.pose` 当前只做透传兼容，不参与世界坐标变换
- 当前最终输出统一是相机坐标系
- 外部系统建议主用 `objects[].object_3d`
- `objects[].pose_camera` 保留给调试和与 `SAM3D` 原始输出做对比
- 当前默认只请求 `mesh`，不请求 `gaussian`
